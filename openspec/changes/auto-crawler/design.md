## Context

目前 UNIQLO 價格監控系統已具備核心爬蟲組件，但只能透過手動腳本或 API 呼叫。為了維持資料庫的數據新鮮度，需要一套背景機制自動執行更新。

## Goals / Non-Goals

**Goals:**
- 實現無人值守的每日價格自動更新。
- 確保伺服器重啟後能自動繼續計時。
- 最小化對官網的負擔（頻率限制）。

**Non-Goals:**
- 提供網頁端設定定時時間的情境（暫時硬編碼於後端）。
- 提供複雜的任務排隊系統（如 Redis/Bull）。

## Decisions

- **工具選擇: `node-cron`**: 相比於簡單的 `setInterval`，`node-cron` 支援標準的 Cron 語法，更適合精確設定凌晨執行任務。
- **執行時間: 每日 01:00**: 避開一般人購物高峰，減少對官網與資料庫的負載壓力。
- **錯誤處理**: 若爬蟲失敗，紀錄錯誤日誌但不中斷下次排程。

## Risks / Trade-offs

- [Risk] → 官網防爬機制升級。
- [Mitigation] → 使用 Puppeteer 隨機化 User-Agent 與間隔。
- [Risk] → 伺服器停機導致跳過某日更新。
- [Mitigation] → 下次啟動時應有一次校檢或補抓邏輯。
